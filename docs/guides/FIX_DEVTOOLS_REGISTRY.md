# План: включить режим разработчика в Chrome и Яндекс.Браузере через реестр

Если плашки «Режим разработчика» неактивны, а в Edge всё работает — политика задаётся в реестре Windows. Ниже пошаговый план.

---

## 1. Открыть реестр

- Нажмите **Win + R**, введите **`regedit`**, нажмите Enter.
- Подтвердите запрос контроля учётной записи (Да), если появится.

---

## 2. Проверить, откуда блокировка (опционально)

В Chrome откройте вкладку **`chrome://policy`**, в Яндексе — **`browser://policy`**.  
Посмотрите, есть ли политика **DeveloperToolsDisabled** и какой у неё источник (часто «Registry: Machine» или «Registry: User»). Это подскажет, что править: HKLM или HKCU.

---

## 3. Исправить политику для Chrome

### Вариант A: политика для всех пользователей (чаще всего)

1. В левой панели реестра откройте по очереди:
   ```
   HKEY_LOCAL_MACHINE
   → SOFTWARE
   → Policies
   → Google
   → Chrome
   ```
2. В правой панели найдите параметр **DeveloperToolsDisabled**.
3. Если он есть:
   - Дважды щёлкните по нему.
   - Значение измените на **0**.
   - ОК.
4. Если параметра нет — ничего не создавайте, переходите к шагу 4 (Яндекс).

### Вариант B: политика только для текущего пользователя

1. Откройте:
   ```
   HKEY_CURRENT_USER
   → SOFTWARE
   → Policies
   → Google
   → Chrome
   ```
2. Если есть **DeveloperToolsDisabled** — поставьте значение **0** (как в варианте A).

---

## 4. Исправить политику для Яндекс.Браузера

### Вариант A: для всех пользователей

1. Откройте:
   ```
   HKEY_LOCAL_MACHINE
   → SOFTWARE
   → Policies
   → YandexBrowser
   ```
2. Если есть **DeveloperToolsDisabled** — поставьте значение **0**.

### Вариант B: для текущего пользователя

1. Откройте:
   ```
   HKEY_CURRENT_USER
   → SOFTWARE
   → Policies
   → YandexBrowser
   ```
2. Если есть **DeveloperToolsDisabled** — поставьте значение **0**.

---

## 5. Удалить параметр вместо изменения (если хотите)

Вместо значения **0** можно просто удалить параметр:

- Правый щелчок по **DeveloperToolsDisabled** → **Удалить** → Да.

Тогда политика перестанет действовать.

---

## 6. Закрыть реестр и перезапустить браузеры

- Закройте **regedit**.
- Полностью закройте Chrome и Яндекс.Браузер (включая фоновые процессы в диспетчере задач, если они есть).
- Запустите браузеры снова.
- Проверьте: **F12** или **Ctrl+Shift+I** — инструменты разработчика должны открываться; плашка «Режим разработчика» в настройках расширений должна стать активной.

---

## Краткая шпаргалка путей

| Браузер | Куст | Путь |
|--------|------|------|
| Chrome | HKLM | `SOFTWARE\Policies\Google\Chrome` |
| Chrome | HKCU | `SOFTWARE\Policies\Google\Chrome` |
| Яндекс | HKLM | `SOFTWARE\Policies\YandexBrowser` |
| Яндекс | HKCU | `SOFTWARE\Policies\YandexBrowser` |

Параметр: **DeveloperToolsDisabled** → значение **0** или удалить.

---

## Если после правок снова блокируется

Значит политику подхватывает что-то извне:

- **Групповые политики (GPO)** — если ПК в домене или настроен через gpedit.msc: в **gpedit.msc** проверьте разделы **Google Chrome** / **Яндекс.Браузер** и отключите политику «Запретить инструменты разработчика» (или аналог).
- **Антивирус/родительский контроль** — отключите на время или проверьте настройки «блокировка настроек браузера».
- **Другое ПО** — могло прописать политику в реестр; после очистки оно может вернуть её при следующем запуске.

В таком случае имеет смысл снова посмотреть **chrome://policy** и **browser://policy** и по источнику политики понять, какая программа или политика её задаёт.
