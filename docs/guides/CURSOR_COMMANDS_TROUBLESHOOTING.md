# Почему не отображаются команды при вводе "/"

## Важно: команды работают только в Chat, не в Composer

**Slash-команды (`/van`, `/plan` и т.д.) в Cursor показываются и работают только в окне Chat (чат), а не в Composer.**

- **Chat:** открой чат (**Ctrl+L** или **Cmd+L**), в поле ввода напиши `/` — должен появиться список команд.
- **Composer:** при вводе `/` в Composer команды **не отображаются** — это ограничение текущей версии Cursor.

Чтобы использовать Memory Bank: открой **Chat** (Ctrl+L), введи `/van` и описание задачи.

---

Cursor берёт команды из двух мест:
- **Проект:** `.cursor/commands/` в корне открытой папки
- **Глобально:** `~/.cursor/commands/` (на Windows: `C:\Users\bbru7\.cursor\commands\`)

Имя команды = имя файла без `.md`. Например: `van.md` → `/van`.

---

## Что проверить, если команды не видны в Chat

### 1. Открыта ли правильная папка

Корнем проекта должна быть **папка «КонтентЗавод»** (та, где лежат `blocks/`, `docs/`, `.cursor/`, `memory-bank/`).

- **Правильно:** File → Open Folder → выбрать `КонтентЗавод`
- **Неправильно:** открыта папка `Desktop` или родительская папка — тогда `.cursor/commands/` не в корне, и проектные команды не подхватятся.

Проверка: в проводнике по пути проекта должна быть папка `.cursor`, а в ней — `commands` с файлами `van.md`, `plan.md` и т.д.

### 2. Перезапуск Cursor

После добавления или изменения файлов в `.cursor/commands/` иногда нужно полностью закрыть Cursor и открыть снова, чтобы список команд обновился.

### 3. Где вводить "/"

Вводить `/` нужно **только в Chat** (Ctrl+L), не в Composer. В Composer список команд по `/` не показывается.

### 4. Cursor в браузере (Cursor Cloud)

В веб-версии Cursor (браузер) **локальные** команды из `.cursor/commands/` репозитория сейчас не поддерживаются. В таком случае:

- используйте **десктопное приложение** Cursor, или  
- скопируйте команды в **глобальную** папку (см. ниже) — они будут доступны и при открытии этого проекта в десктопе.

### 5. Глобальные команды как запасной вариант

Команды продублированы в глобальную папку:

`C:\Users\bbru7\.cursor\commands\`

Там лежат: `van.md`, `plan.md`, `creative.md`, `build.md`, `reflect.md`, `archive.md`.

Они должны появляться при вводе `/` в **любом** проекте в десктопном Cursor. Если в КонтентЗавод команды по-прежнему не видны — проверьте, что при открытии этого проекта видны хотя бы эти глобальные команды (например, `/van`, `/plan`). Если глобальные есть, а проектные нет — значит, Cursor не считает текущую папку корнем с `.cursor/commands/`.

---

## Итог

- Убедись, что открыта папка **КонтентЗавод** и что в ней есть `.cursor/commands/` с `.md` файлами.
- Полностью перезапусти Cursor.
- Вводи `/` в поле чата или Composer.
- Если пользуешься браузером — перейди на десктоп или используй команды из `C:\Users\bbru7\.cursor\commands\`.
- Если после этого команды всё равно не появляются — возможно, баг или ограничение версии Cursor; версию можно посмотреть в Menu → About Cursor.
